# 003 — feat(provider): MockScenario library and rotator

## 개요

기본 random walk는 "조용히 흔들리는 정상 경기"만 만든다. 실제 운영에서 마주치는
**edge case**(갑작스런 마켓 suspend, 후반 골, 경기 연기, odds 폭락)는 안 나온다.
`MockScenario` sealed 계층 네 가지 + `ScenarioRotator`가 그걸 주기적으로 주입해,
시뮬레이터가 "영원히 매끄러운 random walk"처럼 보이지 않게 한다. 부하 테스트는
회전을 끄고 시나리오를 결정론적으로 직접 구동할 수 있다.

## 작업 순서

1. `MockScenario` sealed interface (`id` / `canApply` / `apply`).
2. 네 구현체: `LateGoal` / `MatchPostponed` / `SuddenMarketSuspend` / `OddsCrash`.
3. `MockOddsProvider`에 package-private 접근점 노출 (`emit`, `transitionTo`,
   `activeEvents`).
4. `ScenarioRotator` — `@Scheduled`로 회전, auto-rotate 플래그 존중.
5. 단위 테스트.

## 작업 내역

### MockScenario — sealed + 직접 상태 변이

```java
public sealed interface MockScenario
    permits LateGoal, MatchPostponed, SuddenMarketSuspend, OddsCrash {
  String id();
  boolean canApply(MockOddsProvider.MockEvent event, Instant now);
  void apply(MockOddsProvider.MockEvent event, Instant now, Random rng,
             MockOddsProvider provider);
}
```

**왜 `MockScenarioContext` 같은 추상화를 안 끼웠나** — 시나리오·rotator·provider가
같은 내부 mock 패키지에 있고 production 코드가 안 쓴다. dispatch에 필요한 연산이
넷(상태 읽기, odds 변이, market status 전환, lifecycle 전환)인데, 이걸 인터페이스로
감싸면 테스트성·재사용 이득 없이 간접층만 는다. 두 번째 mock 구현이 생기는 날이
추출 시점이다. (YAGNI를 의식적으로 적용한 결정.)

### 네 시나리오

**LateGoal** — IN_PLAY, endAt 직전 10 mock분 윈도우. selection 하나 가격 50%
폭락. 윈도우 판정은 `event.endAt`에 대한 실제 `Duration` 비교라 시간 scale 압축이
자동 적용된다:

```java
public final class LateGoal implements MockScenario {
  static final Duration TRIGGER_WINDOW_MOCK = Duration.ofMinutes(10);
  private static final double GOAL_ODDS_MULTIPLIER = 0.5;

  public String id() { return "LateGoal"; }

  public boolean canApply(MockOddsProvider.MockEvent event, Instant now) {
    if (event.status != EventLifecycleStatus.IN_PLAY) return false;
    Duration remaining = Duration.between(now, event.endAt);
    return !remaining.isNegative() && remaining.compareTo(TRIGGER_WINDOW_MOCK) <= 0;
  }

  public void apply(MockOddsProvider.MockEvent event, Instant now, Random rng,
                    MockOddsProvider provider) {
    MockOddsProvider.MockMarket market = event.markets.values().iterator().next();
    var selections = List.copyOf(market.selections.values());
    var target = selections.get(rng.nextInt(selections.size()));
    Odds previous = target.currentOdds;
    BigDecimal nextDecimal = previous.decimal()
        .multiply(BigDecimal.valueOf(GOAL_ODDS_MULTIPLIER))
        .max(BigDecimal.valueOf(OddsSimulator.MIN_ODDS))   // 1.01 바닥 보장
        .setScale(Odds.SCALE, RoundingMode.HALF_EVEN);
    Odds next = Odds.ofDecimal(nextDecimal);
    target.currentOdds = next;
    provider.emit(event.summary.eventId(),
        new ProviderEvent.OddsUpdated(
            event.summary.eventId(), market.marketId, target.selectionId, previous, next, now));
  }
}
```

**MatchPostponed** — SCHEDULED만. `transitionTo(POSTPONED)` — 기본 시뮬레이터의
SCHEDULED→IN_PLAY와 같은 경로라 `LifecycleUpdated` 하나 emit (ADR-0012: 연기/취소
경기는 베팅 void):

```java
public final class MatchPostponed implements MockScenario {
  public String id() { return "MatchPostponed"; }
  public boolean canApply(MockOddsProvider.MockEvent event, Instant now) {
    return event.status == EventLifecycleStatus.SCHEDULED;
  }
  public void apply(MockOddsProvider.MockEvent event, Instant now, Random rng,
                    MockOddsProvider provider) {
    provider.transitionTo(event, EventLifecycleStatus.POSTPONED, now);
  }
}
```

**SuddenMarketSuspend** — OPEN 마켓을 SUSPENDED로, reason "simulated in-play
pause (VAR check)". auto-reopen은 일부러 범위 밖(다음 rotator tick 또는 trader
endpoint가 처리):

```java
public final class SuddenMarketSuspend implements MockScenario {
  private static final String REASON = "simulated in-play pause (VAR check)";
  public boolean canApply(MockOddsProvider.MockEvent event, Instant now) {
    if (event.status != EventLifecycleStatus.IN_PLAY
        && event.status != EventLifecycleStatus.SCHEDULED) return false;
    return event.markets.values().stream().anyMatch(m -> m.status == MarketStatus.OPEN);
  }
  public void apply(MockOddsProvider.MockEvent event, Instant now, Random rng,
                    MockOddsProvider provider) {
    var target = event.markets.values().stream()
        .filter(m -> m.status == MarketStatus.OPEN).findFirst().orElseThrow();
    MarketStatus previous = target.status;
    target.status = MarketStatus.SUSPENDED;
    provider.emit(event.summary.eventId(),
        new ProviderEvent.MarketStatusUpdated(
            event.summary.eventId(), target.marketId, previous, MarketStatus.SUSPENDED, REASON, now));
  }
}
```

**OddsCrash** — 아무 때나(IN_PLAY/SCHEDULED) selection 하나를 직전 30%로 폭락.
LateGoal과 같은 폭락 구조지만 시점 무관(레드카드·부상·기권 모델). **1% threshold의
"반드시 publish되는" 정준 케이스** 역할:

```java
public final class OddsCrash implements MockScenario {
  private static final double CRASH_MULTIPLIER = 0.3;
  public boolean canApply(MockOddsProvider.MockEvent event, Instant now) {
    return event.status == EventLifecycleStatus.IN_PLAY
        || event.status == EventLifecycleStatus.SCHEDULED;
  }
  // apply(): LateGoal과 동일 — target selection의 odds를 CRASH_MULTIPLIER배,
  //          MIN_ODDS 바닥, OddsUpdated emit.
}
```

### ScenarioRotator

```java
@Component
@Profile("mock")
public class ScenarioRotator {

  private final MockProperties props;
  private final MockOddsProvider provider;
  private final Clock clock;
  private final List<MockScenario> scenarios;
  private final Random rng;

  public ScenarioRotator(MockProperties props, MockOddsProvider provider, Clock clock) {
    this.props = props;
    this.provider = provider;
    this.clock = clock;
    this.scenarios =
        List.of(new LateGoal(), new MatchPostponed(), new SuddenMarketSuspend(), new OddsCrash());
    // provider와 다른 시드(+1)로 두 random walk가 동조되지 않게.
    this.rng = props.randomSeed() == 0 ? new Random() : new Random(props.randomSeed() + 1);
  }

  @Scheduled(fixedRateString = "${oddsfeed.mock.scenarios.rotation-interval-seconds:60}",
             timeUnit = TimeUnit.SECONDS)
  void scheduledRotate() {
    if (!props.scenarios().autoRotate()) return;   // 부하 테스트는 끄고 직접 구동
    rotateOnce(clock.instant());
  }

  void rotateOnce(Instant now) {     // package-private — 테스트가 스케줄러 없이 한 스텝
    MockScenario scenario = scenarios.get(rng.nextInt(scenarios.size()));
    Optional<MockOddsProvider.MockEvent> target =
        provider.activeEvents().stream().filter(e -> scenario.canApply(e, now)).findFirst();
    target.ifPresent(event -> scenario.apply(event, now, rng, provider));
  }
}
```

무작위 시나리오 1개를 골라 `canApply`가 true인 첫 이벤트에 적용한다. 적용 가능한
이벤트가 없으면 조용히 skip(블로킹 아님).

## 요약

- 네 edge-case 시나리오를 sealed 계층으로.
- 직접 상태 변이(추상화 의도적 생략 — YAGNI).
- auto-rotate 플래그로 부하 테스트는 결정론적 구동.
- `advance()`에 POSTPONED 정지 추가.
