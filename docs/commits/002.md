# 002 — feat(provider): MockOddsProvider with deterministic event simulator

## 개요

`OddsProvider`의 **기본 구현체**. `@Profile("mock")`로 활성화되고, 외부 키 없이
로컬에서 전체 이벤트 lifecycle을 굴린다. 부하 테스트·통합 테스트·카오스
시나리오가 전부 이 위에서 돈다(ADR-0010 — mock 1차). 핵심은 **결정론
(deterministic)** — 같은 seed면 같은 결과라 테스트가 재현 가능하다.

## 작업 순서

1. `ApplicationConfig` — `@EnableScheduling` + `Clock` @Bean.
2. `MockProperties` — `oddsfeed.mock.*` 바인딩.
3. `OddsSimulator` — 순수 가격 random walk (Spring 무관).
4. `MockOddsProvider` — lifecycle 진행 + odds emit + 결과 합성.
5. 단위 테스트 — lifecycle, emit, 결과, 시뮬레이터 수렴/클램핑/결정성.

## 작업 내역

### OddsSimulator — 순수 함수로 분리

가격 모델을 Spring·스레드와 무관한 순수 클래스로 떼냈다. 테스트가 컨테이너 없이
바로 돈다. 상수 4개 + `initialOdds`(공정값) + `nextOdds`(한 스텝) 전부:

```java
public final class OddsSimulator {

  static final double NOISE_STDDEV = 0.02;     // 2% 곱셈 노이즈
  static final double MEAN_REVERSION = 0.10;   // fair 쪽으로 10% 당김
  static final double MIN_ODDS = 1.01;
  static final double MAX_ODDS = 100.0;

  private OddsSimulator() {}

  /** 신규 selection 초기 odds: 정확히 공정값 (V1은 margin 0). */
  public static Odds initialOdds(double impliedProbability) {
    double fair = 1.0 / impliedProbability;
    return Odds.ofDecimal(BigDecimal.valueOf(fair).setScale(Odds.SCALE, RoundingMode.HALF_EVEN));
  }

  // fair odds = 1/p. 곱셈 노이즈로 흔든 뒤 fair 쪽으로 일부 평균회귀.
  public static Odds nextOdds(Odds current, double impliedProbability, Random rng) {
    double fair = 1.0 / impliedProbability;
    double cur = current.decimal().doubleValue();
    double noisy = cur * (1.0 + rng.nextGaussian() * NOISE_STDDEV);
    double next = noisy * (1.0 - MEAN_REVERSION) + fair * MEAN_REVERSION;
    double clamped = Math.max(MIN_ODDS, Math.min(MAX_ODDS, next));     // [1.01, 100]
    return Odds.ofDecimal(BigDecimal.valueOf(clamped).setScale(Odds.SCALE, RoundingMode.HALF_EVEN));
  }
}
```

- **평균회귀(mean reversion)** 가 핵심. 순수 random walk는 발산하거나 1.0 밑으로
  내려가 `Odds`(>= 1.0만 허용)가 예외를 던진다. fair 쪽으로 10%씩 당기고
  `[1.01, 100]`으로 clamp해서 항상 유효한 odds만 나오게 했다.
- base 확률: HOME 0.45 / DRAW 0.25 / AWAY 0.30 (합 1.0, **margin 0** — V1).

### MockOddsProvider — 상태 + 시간 진행 + emit

상태는 `EventId` 키의 동시성 맵 3개 — 가변 `MockEvent`, orchestrator가 구독하는
multicast `Sinks.Many`, FINISHED 때 채워지는 `MatchOutcome`. 변이는 스케줄러
스레드에서만:

```java
@Component
@Profile("mock")
public class MockOddsProvider implements OddsProvider {

  static final int INITIAL_EVENT_COUNT = 3;
  static final Duration MATCH_DURATION_MOCK = Duration.ofMinutes(90);
  static final Duration KICKOFF_SPACING_MOCK = Duration.ofMinutes(1);
  private static final double SECONDS_PER_MINUTE = 60.0;

  private final MockProperties props;
  private final Clock clock;
  private final Map<EventId, MockEvent> events = new ConcurrentHashMap<>();
  private final Map<EventId, Sinks.Many<ProviderEvent>> streams = new ConcurrentHashMap<>();
  private final Map<EventId, MatchOutcome> outcomes = new ConcurrentHashMap<>();
  private Random rng;

  public MockOddsProvider(MockProperties props, Clock clock) {
    this.props = props;
    this.clock = clock;
  }

  @PostConstruct
  void seed() {
    rng = props.randomSeed() == 0 ? new Random() : new Random(props.randomSeed());
    Instant now = clock.instant();
    for (int i = 0; i < INITIAL_EVENT_COUNT; i++) {
      Instant kickoff = now.plus(toRealDuration(KICKOFF_SPACING_MOCK.multipliedBy(i)));
      Instant end = kickoff.plus(toRealDuration(MATCH_DURATION_MOCK));
      MockEvent event = buildEvent(kickoff, end, i, now);   // EPL 3경기 seed
      events.put(event.summary.eventId(), event);
      streams.put(event.summary.eventId(), Sinks.many().multicast().onBackpressureBuffer());
    }
  }
```

tick은 `@Scheduled`가 부르지만, 테스트는 스케줄러를 우회해 package-private
`tick(Instant)`를 직접 부른다:

```java
@Scheduled(fixedRateString = "${oddsfeed.mock.tick-interval-ms:500}")
void scheduledTick() {
  tick(clock.instant());
}

void tick(Instant now) {            // package-private — 테스트가 직접 호출
  for (MockEvent event : events.values()) {
    advance(event, now);
  }
}
```

`advance`가 이 커밋의 심장 — 한 이벤트의 lifecycle 전이 + (OPEN 마켓에 한해)
selection별 random walk emit:

```java
private void advance(MockEvent event, Instant now) {
  if (event.status == EventLifecycleStatus.FINISHED
      || event.status == EventLifecycleStatus.CANCELLED) {
    return;
  }
  if (event.status == EventLifecycleStatus.SCHEDULED && !now.isBefore(event.kickoffAt)) {
    transitionTo(event, EventLifecycleStatus.IN_PLAY, now);
  }
  if (event.status == EventLifecycleStatus.IN_PLAY && !now.isBefore(event.endAt)) {
    transitionTo(event, EventLifecycleStatus.FINISHED, now);
    outcomes.put(event.summary.eventId(), synthesizeOutcome(event, now));
    return;
  }
  for (MockMarket market : event.markets.values()) {
    if (market.status != MarketStatus.OPEN) {
      continue;
    }
    for (MockSelection selection : market.selections.values()) {
      Odds previous = selection.currentOdds;
      Odds next = OddsSimulator.nextOdds(previous, selection.impliedProbability, rng);
      if (!previous.equals(next)) {                 // 변화 없으면 emit 안 함
        selection.currentOdds = next;
        emit(event.summary.eventId(),
            new ProviderEvent.OddsUpdated(
                event.summary.eventId(), market.marketId, selection.selectionId,
                previous, next, now));
      }
    }
  }
}
```

전이는 status를 바꾸고 불변 `EventSummary`를 새로 만들어 `LifecycleUpdated`를
emit, 결과 합성은 base 확률로 score를 굴린다:

```java
private void transitionTo(MockEvent event, EventLifecycleStatus next, Instant now) {
  event.status = next;
  event.summary = new EventSummary(
      event.summary.eventId(), event.summary.sport(), event.summary.competition(),
      event.summary.homeTeam(), event.summary.awayTeam(),
      event.summary.scheduledStartAt(), next);
  emit(event.summary.eventId(),
      new ProviderEvent.LifecycleUpdated(
          event.summary.eventId(), next, event.summary.scheduledStartAt(), now));
}

private void emit(EventId eventId, ProviderEvent event) {
  Sinks.Many<ProviderEvent> sink = streams.get(eventId);
  if (sink != null) {
    sink.tryEmitNext(event);
  }
}

private MatchOutcome synthesizeOutcome(MockEvent event, Instant now) {
  double roll = rng.nextDouble();
  String score;
  if (roll < props.baseHomeWinProbability()) {
    score = "2-1";
  } else if (roll < props.baseHomeWinProbability() + props.baseDrawProbability()) {
    score = "1-1";
  } else {
    score = "0-1";
  }
  return new MatchOutcome(event.summary.eventId(), score, MatchFinalStatus.COMPLETED, Map.of(), now);
}
```

> **테스트 가능성.** `Clock`을 주입한다(@Bean은 system UTC, 테스트는
> `Clock.fixed`). `@Scheduled` 메서드는 `clock.instant()`를 읽지만, 테스트는
> 스케줄러를 우회하고 package-private `tick(Instant)`를 직접 부른다. 그래서
> `@EnableScheduling`이 테스트 클래스패스에 안 들어와도 된다. C/C++에서 시간
> 의존 코드를 테스트할 때 `now()`를 함수 포인터로 주입하던 것과 같은 발상.

`OddsProvider` 3메서드 구현과 시간 압축 변환 — `toRealDuration`이 **결과 섹션의
산수 버그**가 났던 자리다:

```java
private Duration toRealDuration(Duration mockDuration) {
  double mockMinutes = mockDuration.toSeconds() / SECONDS_PER_MINUTE;
  long realSeconds = Math.max(1L, (long) (mockMinutes / props.minutesPerSecond()));
  return Duration.ofSeconds(realSeconds);
}

@Override public List<EventSummary> listEvents(Sport sport) {
  List<EventSummary> result = new ArrayList<>();
  for (MockEvent event : events.values()) {
    if (event.summary.sport() == sport) result.add(event.summary);
  }
  return List.copyOf(result);
}
@Override public Flux<ProviderEvent> streamEvents(EventId eventId) {
  Sinks.Many<ProviderEvent> sink = streams.get(eventId);
  return sink == null ? Flux.empty() : sink.asFlux();
}
@Override public Optional<MatchOutcome> getMatchResult(EventId eventId) {
  return Optional.ofNullable(outcomes.get(eventId));
}
```

내부 가변 상태는 package-private 정적 클래스 3종으로 묶었다:

```java
static final class MockEvent {
  EventSummary summary; final Map<MarketId, MockMarket> markets = new ConcurrentHashMap<>();
  Instant kickoffAt; Instant endAt; EventLifecycleStatus status;
}
static final class MockMarket {        // 생성 시 status = OPEN
  final MarketId marketId; final MarketType type; MarketStatus status; final Map<SelectionId, MockSelection> selections;
}
static final class MockSelection {
  final SelectionId selectionId; final String name; final double impliedProbability; Odds currentOdds;
}
```

### V1 scope + 설정 바인딩

- 이벤트당 `MATCH_RESULT_1X2` 마켓 1개 + selection 3개(HOME/DRAW/AWAY).
- 시나리오 회전·마켓 suspend·late goal은 **없음** — 다음 커밋(003)의
  `MockScenario`. 단 `MockProperties.Scenarios`에 auto-rotate 플래그는 미리
  넣어둬서 003이 이 클래스 모양을 안 건드리게 했다.

```java
@ConfigurationProperties(prefix = "oddsfeed.mock")
public record MockProperties(
    double minutesPerSecond, Scenarios scenarios,
    double baseHomeWinProbability, double baseDrawProbability, double baseAwayWinProbability,
    long randomSeed, int tickIntervalMs) {
  public record Scenarios(boolean autoRotate, int rotationIntervalSeconds) {}
}

@Configuration
@EnableScheduling
@EnableConfigurationProperties(MockProperties.class)
public class ApplicationConfig {
  @Bean public Clock systemClock() { return Clock.systemUTC(); }
}
```

**시간 scale**: `minutesPerSecond`로 mock시간을 압축. 기본 1(90분 경기가 90초),
부하 테스트는 60(roster가 몇 초마다 순환).

## 결과

19개 테스트 통과. **여기서 산수 버그를 하나 잡았다** — `toRealDuration`이 mock분
→ 실초 변환에서 60을 한 번 더 곱해 90분 경기가 5400초가 됐다.
`tickProgressesEventThroughFullLifecycle`가 FINISHED 도달 실패로 빨갛게 떠서
즉시 발견·수정. 단위 테스트가 "조용한 산수 실수"를 잡은 정확한 사례다.

## 요약

- `OddsSimulator`: fair odds 중심 평균회귀 random walk + clamp → 항상 유효 odds.
- `MockOddsProvider`: `@Scheduled` tick으로 lifecycle 진행, 이벤트별 Sink emit.
- `Clock` 주입 + package-private `tick`으로 스케줄러 없이 테스트.
- 시간 scale로 부하 테스트 시 빠른 lifecycle 재현.
