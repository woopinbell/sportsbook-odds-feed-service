# 013 — fix(mock): 경기 종료 시 완전 채점된 MatchResult 발행

## 개요

dev 단계가 아니라 **Phase 5**에서 "완료 경기가 정산되지 않는" 원인으로 드러난 결함이다. mock은 최종 score를 합성했지만 MatchResult의 `resultDetail`을 **빈 맵**으로 발행했다. settlement은 selection별 WON/LOST 계약(`resultDetail`)을 stamp해 정산하는데 그게 비어 있으니 완료된 경기가 정산되지 않았다(payout 0, outbox 0). 더해 순서 버그도 있었다: outcome을 저장하기 *전에* FINISHED lifecycle을 emit해서, 그걸 동기로 처리하는 orchestrator가 곧장 `getMatchResult`를 부르면 빈 결과를 받았다.

## 작업 순서

1. outcome을 저장한 뒤 FINISHED를 emit(순서 교정).
2. 합성된 승자에 맞춰 각 1X2 selection을 채점(WON / LOST).

## 작업 내역

### outcome-before-emit 순서

`transitionTo(FINISHED)`는 LifecycleUpdated(FINISHED)를 emit하고, orchestrator가 이를 **동기로** 처리하며 즉시 `getMatchResult`를 부른다. outcome이 아직 map에 없으면 빈 결과 → MatchResult 미발행 → settlement 미트리거. `advance`의 FINISHED 분기에서 두 줄의 **순서를 뒤집었다**:

```java
if (event.status == EventLifecycleStatus.IN_PLAY && !now.isBefore(event.endAt)) {
  // 순서 교정: outcome을 먼저 저장한 뒤 emit.
  outcomes.put(event.summary.eventId(), synthesizeOutcome(event, now));   // 먼저 저장
  transitionTo(event, EventLifecycleStatus.FINISHED, now);                // 그다음 emit
  return;
}
```

### 1X2 채점 (resultDetail 계약 채우기)

score로 승자(HOME / DRAW / AWAY)를 정하고, `gradeSelections`가 각 selection을 승자면 WON·나머지는 LOST로 매핑해 `selectionId → SettlementResult.name()` 맵을 만든다. 이것이 settlement이 consume하는 계약이다(빈 `Map.of()`가 숨기던 결함):

```java
private MatchOutcome synthesizeOutcome(MockEvent event, Instant now) {
  double roll = rng.nextDouble();
  String score;
  String winningSelection;
  if (roll < props.baseHomeWinProbability()) {
    score = "2-1"; winningSelection = "HOME";
  } else if (roll < props.baseHomeWinProbability() + props.baseDrawProbability()) {
    score = "1-1"; winningSelection = "DRAW";
  } else {
    score = "0-1"; winningSelection = "AWAY";
  }
  return new MatchOutcome(event.summary.eventId(), score, MatchFinalStatus.COMPLETED,
      gradeSelections(event, winningSelection), now);     // 이전: Map.of() (빈 맵)
}

private Map<String, String> gradeSelections(MockEvent event, String winningSelection) {
  Map<String, String> detail = new LinkedHashMap<>();
  for (MockMarket market : event.markets.values()) {
    for (MockSelection selection : market.selections.values()) {
      SettlementResult result =
          selection.name.equals(winningSelection) ? SettlementResult.WON : SettlementResult.LOST;
      detail.put(selection.selectionId.value().toString(), result.name());
    }
  }
  return detail;
}
```

## 결과

- 완료 경기가 채점된 MatchResult를 발행 → settlement이 selection별로 정산. 라이브 검증: 한 이벤트 3 selection에 single 3건 → 1 WON / 2 LOST.
- 테스트: `finishedEventGradesEachSelectionAgainstScore`(detail 3건, 1 WON·2 LOST, 채점된 승자가 score와 일치).

## 요약

- 빈 `resultDetail` → settlement이 stamp할 계약이 없어 완료 경기 미정산(버그 #6).
- outcome을 emit 전에 저장(순서 버그 #4) — orchestrator가 FINISHED를 동기 처리 후 즉시 getMatchResult.
- score → 승자 → 각 1X2 selection WON/LOST 채점. settlement의 resultDetail 계약을 cross-repo로 완성.

## 기억·설명 Level

- **L1**: 빈 resultDetail → 채점된 맵, outcome을 emit 전에 저장.
- **L2**: orchestrator가 FINISHED를 동기 처리하며 getMatchResult를 부르는 순서 의존성, `gradeSelections`의 1X2 매핑.
- **L3**: settlement의 "결과 판정 입력 계약"(selectionId→결과)을 odds-feed가 채우는 cross-repo 완성. 빈 맵이 단위 테스트(score/status만 단언)를 통과하고 full-stack 정산에서만 드러난 점, 그리고 동기 emit→getMatchResult 순서 버그.
