## 작업 내역

### TBD → 실측

| 시나리오 | 목표 | 박제값 |
|---|---|---|
| Publisher → Kafka throughput | 10만/sec | **약 290k/sec** (3배) |
| 1% threshold 정확성 | 100% | **100%** (`OddsFeedPublisherTest` 7케이스) |
| mock → Kafka e2e p99 | < 100ms | TBD (이유 명시) |
| HTTP read p99 | < 50ms | TBD (이유 명시) |

두 TBD는 그냥 비워두지 않고 **왜 아직 못 쟀는지**를 적었다 — e2e는 별도 프로세스
상관이 필요(orchestration Phase 5), HTTP는 실행 인스턴스 + k6 바이너리 필요.
"안 한 것"과 "못 한 것"을 구분해 정직하게.

### caveats 단락

- in-process broker는 JVM ceiling이지 production 단일 broker 수치가 아님.
- Testcontainers cp-kafka가 ARM macOS에서 기동 실패 → EmbeddedKafkaZKBroker.
- e2e latency는 mock·broker 별도 프로세스 상관이 필요.
