# 013 — fix(mock): 경기 종료 시 완전 채점된 MatchResult 발행

## 개요

dev 단계가 아니라 **Phase 5**에서 "완료 경기가 정산되지 않는" 원인으로 드러난 결함이다. mock은 최종 score를 합성했지만 MatchResult의 `resultDetail`을 **빈 맵**으로 발행했다. settlement은 selection별 WON/LOST 계약(`resultDetail`)을 stamp해 정산하는데 그게 비어 있으니 완료된 경기가 정산되지 않았다(payout 0, outbox 0). 더해 순서 버그도 있었다: outcome을 저장하기 *전에* FINISHED lifecycle을 emit해서, 그걸 동기로 처리하는 orchestrator가 곧장 `getMatchResult`를 부르면 빈 결과를 받았다.

## 작업 순서

1. outcome을 저장한 뒤 FINISHED를 emit(순서 교정).
2. 합성된 승자에 맞춰 각 1X2 selection을 채점(WON / LOST).

## 작업 내역

### outcome-before-emit 순서

`advance`의 FINISHED 분기에서 두 줄의 순서가 의미를 가진다:

```java
if (event.status == EventLifecycleStatus.IN_PLAY && !now.isBefore(event.endAt)) {
  // TODO: 순서가 핵심 — outcome을 map에 먼저 저장한 뒤 transitionTo(FINISHED).
  //       (왜? transitionTo가 emit하는 FINISHED를 orchestrator가 어떻게 처리하나)
  return;
}
```

**책임:** `transitionTo(FINISHED)`가 emit하는 LifecycleUpdated를 orchestrator가
**동기로** 처리하며 즉시 `getMatchResult`를 부른다. outcome이 아직 없으면 빈
결과 → MatchResult 미발행 → settlement 미트리거.
