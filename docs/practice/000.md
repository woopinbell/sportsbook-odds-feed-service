# 000 — chore(project): initialize odds-feed-service layout

## 개요

odds-feed-service의 골격을 세운 첫 커밋. 비즈니스 로직은 0이고, **빌드 정의 +
profile + 관측성 wiring + 빈 패키지 레이아웃**만 깐다. 이 repo는 Phase 2 leaf —
`shared-protocol`만 의존하고, betting-service가 Redis로 odds를 읽기 때문에
betting보다 먼저 와야 한다.

핵심 결정 하나: **이 repo는 라이브러리가 아니라 application이다.** 그래서
`shared-protocol`처럼 BOM만 import하는 게 아니라 `spring-boot-starter-parent`를
**상속(parent)** 한다. application은 실행 가능한 fat jar와 `spring-boot:run`이
필요하고, 그걸 `spring-boot-maven-plugin`이 해주는데 그 plugin 관리가 parent에
들어 있다.

       무엇을 하는가"와 1:1이어야 한다. 최소: shared-protocol / REST 서빙 /
       WebClient(real provider 전용) / Redis 캐시 / Kafka publish / actuator /

<build><plugins>
  <!-- TODO: 빌드 plugin 3종을 verify 게이트로. (1) 실행 fat jar + spring-boot:run,
