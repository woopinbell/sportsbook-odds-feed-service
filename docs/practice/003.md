# 003 — feat(provider): MockScenario library and rotator

## 개요

기본 random walk는 "조용히 흔들리는 정상 경기"만 만든다. 실제 운영에서 마주치는
**edge case**(갑작스런 마켓 suspend, 후반 골, 경기 연기, odds 폭락)는 안 나온다.
`MockScenario` sealed 계층 네 가지 + `ScenarioRotator`가 그걸 주기적으로 주입해,
시뮬레이터가 "영원히 매끄러운 random walk"처럼 보이지 않게 한다. 부하 테스트는
회전을 끄고 시나리오를 결정론적으로 직접 구동할 수 있다.

## 작업 순서

1. `MockScenario` sealed interface (`id` / `canApply` / `apply`).
2. 네 구현체: `LateGoal` / `MatchPostponed` / `SuddenMarketSuspend` / `OddsCrash`.
3. `MockOddsProvider`에 package-private 접근점 노출 (`emit`, `transitionTo`,
   `activeEvents`).
4. `ScenarioRotator` — `@Scheduled`로 회전, auto-rotate 플래그 존중.
5. 단위 테스트.

## 작업 내역

### MockScenario — sealed + 직접 상태 변이

계약이라 전부 노출:

```java
public sealed interface MockScenario
    permits LateGoal, MatchPostponed, SuddenMarketSuspend, OddsCrash {
  String id();
  boolean canApply(MockOddsProvider.MockEvent event, Instant now);
  void apply(MockOddsProvider.MockEvent event, Instant now, Random rng, MockOddsProvider provider);
}
```

**책임:** 4종을 sealed로 봉인. `MockScenarioContext` 같은 추상화를 일부러 안
끼운 이유(YAGNI — 같은 패키지·test-only·두 번째 구현 없음)를 설명할 수 있어야.
