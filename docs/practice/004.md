# 004 — feat(provider): TheOddsApiProvider for the real upstream

## 개요

`OddsProvider`의 **real 구현**(`@Profile("real")`). The Odds API v4를 WebClient로
폴링한다. 무료 tier(500 req/월, 5 req/분 권장)를 chatty한 poll 루프가 태워버리지
않도록 **sliding-window RateLimiter**(분당)와 **Redis 월간 quota 카운터**가 둘 다
바깥 호출을 막는다. mock과 같은 인터페이스라 orchestrator는 profile을 모른다.

## 작업 순서

1. `pom.xml`에 WireMock(test) 추가.
2. `RealProperties` — `oddsfeed.real.*` 바인딩.
3. `TheOddsApiDtos` — v4 응답 wire 모델.
4. `RateLimiter` (sliding window) + `QuotaCounter` 인터페이스 + `RedisQuotaCounter`.
5. `TheOddsApiProvider` — poll + diff + emit.
6. `TheOddsApiClientConfig` — WebClient·RateLimiter @Bean.
7. 테스트 — RateLimiter 단위 + WireMock 기반 provider.

## 작업 내역

### wire 모델 — snake_case → camelCase

외부 API 스펙 계약이라 노출. 매핑 전략(`@JsonNaming`)의 의미를 설명할 수 있어야:

```java
public final class TheOddsApiDtos {
  private TheOddsApiDtos() {}

  @JsonNaming(SnakeCaseStrategy.class)
  public record Event(String id, String sportKey, String sportTitle,
      Instant commenceTime, String homeTeam, String awayTeam, List<Bookmaker> bookmakers) {}
  @JsonNaming(SnakeCaseStrategy.class)
  public record Bookmaker(String key, String title, Instant lastUpdate, List<Market> markets) {}
  @JsonNaming(SnakeCaseStrategy.class)
  public record Market(String key, Instant lastUpdate, List<Outcome> outcomes) {}
  @JsonNaming(SnakeCaseStrategy.class)
  public record Outcome(String name, BigDecimal price) {}
}
```

**책임:** 클래스 레벨 `@JsonNaming`으로 Java는 camelCase, Jackson은 업스트림이
보내는 snake_case(`sport_key`)를 읽는다. DTO는 real 패키지 밖으로 안 나가고
경계에서 `EventSummary`/`OddsUpdated`로 변환된다.
