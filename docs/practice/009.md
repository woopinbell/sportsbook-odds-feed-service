# 009 — test(load): record kafka publish throughput baseline

## 개요

`KafkaPublishThroughputTest`로 publisher → Kafka **지속 처리량**을 측정한다.
sportsbook/CLAUDE.md의 Level 2 목표는 **10만 events/sec**. 이 커밋은 그 숫자를
실제로 재서 박제한다.

## 작업 순서

1. `load-test/README.md` + `scenarios/odds_read.js`(k6 골격).
2. `KafkaPublishThroughputTest` — in-process broker 처리량.
3. 3회 실행 → `results/2026-05-28/BASELINE.md` + `results/BEST.md`.

**책임:** 매 send가 1% threshold를 넘겨야 필터가 측정을 지배 안 함. warm-up은

같은 회귀 안전망 — 실제 숫자는 결과 파일로.
